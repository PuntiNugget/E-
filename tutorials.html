Introduction To E#

In my humble opinion, this is one of the easiest languages out there, so it should be easy to grasp.

So the first thing you should do is get rid of all of the example code. Nothing should be there, nada!

Now that your canvas is empty, let's write our very first line of code to display some text on the screen.

Program.e# E# Script 
set string greeting to "Welcome to E#!"
    draw text greeting at x 100 y 150

If you run this program, you should see your greeting appear right in the middle of the game canvas!

This is the most basic program, or at least one of the most basic programs you can write.

Now, it is tempting to start making a game immediately. Trust me I understand that urge. But we first have to learn the engine.

Learning the UI

The Code Editor

Look at the left hand side

This is the code editor. You've already interacted with it before! It's where you put in all of your code.

Theres 3 parts to the editor. First we have the suggestion bar. The first bar you'll see when you look on the bottom is the suggestion bar. This is where simple keywords you want to code will show up. Then press "TAB" and you'll paste that keyword straight in!

Next we have the syntax bar. This will show you anything, and everything that's wrong with you code. HOWEVER! There are such errors called runtime errors that won't show themselves unless you run the program. Sadly, due to being confined in a browser, we cannot add those to the syntax bar. Rather, we have to debug it ourselves

The Game Canvas

Look at the top right corner

This is the game canvas.

We've also already explored this area of the engine. It is where all of your drawn output is

Key word here is drawn, just printing output will cause the program to put it in the terminal. Which is usually not what you want.
